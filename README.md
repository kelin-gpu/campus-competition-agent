# campus-competition-agent

本校竞赛与活动信息服务 AI 助手，面向南京大学学生聚合校外竞赛、校内竞赛与校内活动信息，提供结构化检索、个性化推荐、DDL 提醒和政策相关性解释。

项目定位于 EL 大赛 × 火山引擎 AI 智能体创新专项组，优先使用火山引擎扣子（Coze）智能体、扣子数据库、扣子知识库与 Python 数据采集能力完成 MVP。

## 项目目标

学生获取竞赛和校园活动信息时，常常需要同时翻阅学校官网、学院通知、公众号、QQ群、五育平台、志愿平台和全国竞赛平台。本项目希望把这些分散信息整理成可检索、可推荐、可提醒的结构化数据，并通过对话式智能体提供一站式入口。

MVP 的一句话目标：

> 面向健雄书院及相关专业学生，构建一个可以查询、推荐、提醒和解释竞赛/活动信息的 AI 助手；第一版优先保证数据准确、流程闭环、演示稳定。

## 核心能力

- 自然语言查询：支持询问近期竞赛、校内竞赛、五育活动、快截止项目等。
- 个性化推荐：根据年级、专业、兴趣、目标、类型偏好和截止时间进行推荐排序。
- DDL 提醒：筛选未来 3 / 7 / 14 天内截止的竞赛和活动，并按剩余时间展示。
- 三类信息分组：回答时区分校外竞赛、校内竞赛、校内活动，避免混成普通通知列表。
- 项目简介与推荐理由：每条结果包含简介、适合对象、推荐理由、截止时间、来源和可信度。
- 通知结构化解析：支持从 PDF、网页、公众号链接、QQ 文本或人工文本中抽取结构化字段。
- 溯源与解释：保留原文、来源链接、更新时间和可信度，降低大模型幻觉风险。

## MVP 边界

第一版保留：

- 竞赛/活动自然语言查询
- 基于用户画像的推荐
- 快截止信息提醒
- 校外竞赛、校内竞赛、校内活动分组展示
- 通知文本结构化抽取
- 扣子数据库与知识库联动

第一版暂不做：

- 自动报名
- 精确计算保研、综测或五育加分
- 未授权的个人隐私数据采集
- 未经群管理员授权的 QQ 群实时监听

政策相关性只输出“明确相关 / 可能相关 / 不明确”，最终以学校、学院和活动官方政策为准。

## 数据来源策略

项目采用“权威底库 + 官网定向爬取 + 校内平台接入 + 半自动通知解析 + 人工样本兜底”的组合方式。

优先级建议：

| 优先级 | 数据源 | 采集方式 | 用途 |
| --- | --- | --- | --- |
| P0 | 教育部/高教学会认可竞赛目录 PDF | PDF 解析 + 手动校验 | 建立校外竞赛基础库 |
| P0 | 人工整理样本 | 表格录入 / 批量导入 | 保证 MVP 演示稳定 |
| P1 | 重点竞赛官网 | 定向爬取 | 获取报名时间、比赛时间、规则和报名入口 |
| P1 | 学校官网、教务处、本科生院、学院/书院官网 | 定向爬取 | 获取本校竞赛、活动和政策通知 |
| P1 | 五育平台 | 授权爬取 / 接口接入 | 识别五育活动、类别、报名时间和认定提示 |
| P2 | 赛氪等热门竞赛平台 | 热门项目采集 | 补充校外竞赛候选池 |
| P2 | 公众号、QQ 群历史通知 | 公开链接解析 / 人工提交 / 授权导出 | 补充碎片化校内信息 |

统一处理流程：

```text
原始来源
→ 获取正文/链接/PDF
→ 文本清洗
→ AI 信息抽取
→ 类型判断
→ 标签生成
→ 去重检查
→ 可信度判断
→ 审核/入库
→ 用户端查询与推荐
```

## 推荐与回答逻辑

用户问题进入智能体后，系统优先抽取以下信息：

- 年级
- 专业或方向
- 目标，例如保研、综测、五育、志愿、竞赛经历
- 偏好类型，例如校外竞赛、校内竞赛、活动、讲座
- 时间范围，例如本周、未来 7 天、快截止

推荐排序参考因素：

- 是否未截止
- 专业相关性
- 年级适配度
- 政策相关性
- 竞赛或活动价值
- 截止时间紧急程度
- 来源可信度

回答输出建议按以下结构：

```text
根据你的年级、专业和目标，我筛选了未截止、与你较相关的信息，并按“校外竞赛、校内竞赛、校内活动”分组展示。

一、校外竞赛
1. 项目名称
- 简介：
- 推荐理由：
- 截止时间：
- 政策相关性：
- 来源与更新时间：
```

## Coze 实施路线

建议的 MVP 技术组合：

- 扣子智能体：负责对话入口、用户意图识别、工具调用和最终回答生成。
- 扣子数据库：存储结构化竞赛/活动记录，例如 `event_info`、`source_info`、`raw_document`。
- 扣子知识库：保存原始通知、竞赛章程、政策说明和竞品分析材料，用于 RAG 解释。
- Python 数据采集：解析 PDF、抓取官网通知、清洗文本、批量导入样本数据。
- 审核流程：对自动抽取结果进行人工校验，避免错误信息直接进入用户回答。

推荐最小数据表：

| 表名 | 作用 |
| --- | --- |
| `event_info` | 存储校外竞赛、校内竞赛和校内活动的结构化记录 |
| `source_info` | 管理来源、可信度、采集方式和最近更新时间 |
| `raw_document` | 保存 PDF、网页、公众号、QQ 导出、人工文本等原始材料 |
| `user_profile` | 预留用户画像字段，支持后续个性化推荐 |
